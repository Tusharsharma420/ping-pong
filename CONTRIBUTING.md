# Contributing Guidelines

First off, thank you for considering contributing!

## Branch Naming Standard
- eat/feature-name
- ix/bug-name
- docs/documentation-update

## Pull Request Process
1. Fork the Project
2. Create your Feature Branch (git checkout -b feat/AmazingFeature)
3. Commit your Changes using Conventional Commits (git commit -m 'feat: Add some AmazingFeature')
4. Push to the Branch (git push origin feat/AmazingFeature)
5. Open a Pull Request using the provided PR template.

## Issues
Please use the provided Issue Templates for submitting bug reports or feature requests.
